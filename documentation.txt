PROJECT: MINISHELL

1. Project Overview
MiniShell is a simplified Unix shell implemented in C.
It replicates the core functionalities of a real Linux terminal such as:
Executing external commands.
Handling foreground and background processes.
Supporting pipes.
Job control using signals.
Built-in commands.
Customizable prompt.
The objective of this project is to understand process management, signal handling, and inter-process communication in Linux.

2. Features Implemented
1.External Command Execution:
Extracts available external commands from system.
Executes commands using fork() and execvp().
Supports arguments (e.g., ls -l).

2.Built-in Commands:
Implemented internal commands without using exec:
cd,pwd,exit,echo,jobs,fg,bg
PS1 (custom prompt)

3.Piping (|):
Supports multiple commands connected with pipe.
Uses pipe(), dup2(), and multiple fork() calls.
Example:
ls -l | wc

4.Background Execution:
Allows commands to run in background.
Example:
sleep 30 
Parent does not wait.
Displays background process ID.

5.Job Control:
Foreground Process Handling
Ctrl + C → Terminates foreground process (SIGINT)
Ctrl + Z → Stops foreground process (SIGTSTP)
Background Process Handling
Maintains stopped processes using a linked list.
jobs → Lists stopped jobs
fg → Brings job to foreground
bg → Resumes job in background

6.Signal Handling:
Handled signals:SIGINT,SIGTSTP,SIGCHLD
Ensures:
No zombie processes
Proper process termination
Clean prompt display

3. Data Structures Used
Singly Linked List (Slist)
Used to:
Store stopped background processes.
Manage job control operations (fg, bg, jobs).
Structure contains:
Process ID (PID)
Command name
Next pointer

4.System Calls Used:
fork(),execvp(),wait(), waitpid(),pipe(),dup2(),kill(),signal()

5.Key Learning Outcomes
Through this project, I learned:
Process creation and management in Linux
How shells execute commands internally
Signal handling and job control
Inter-process communication using pipes
Memory management in C
Designing modular and structured C programs
